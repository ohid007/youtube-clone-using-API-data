Different types of variables

Javascript is a dynamically typed variable; we don't need to declare the type of the variable during its declaration. Javascript uses some keywords to declare a variable, such as Var, let, and const.Variables declared using var, let, and const have quite different scopes in JavaScript. It is very necessary to understand these scope rules, Let's discuss these scope rules and functions:

Var:The JavaScript keyword var is used to declare variables that are function scoped. All of the JavaScript keywords were declared using the only "var" keyword prior to the introduction of ES6.


  • Function Scope: variable declared with Var is function scoped means it will be available only inside the function if it's declared within the function. it it's declared outside of function means global declaration. it will be available throughout the script in this case.

  • Hoisting: All the variables declared with Var will be hoisted at the top of the containing function or global scope. Most important thing is only the declaration will be hoisted at the top not the values of that variable.

Let: The let Keyword was introduced in Es6. it is re-assignable, block scoped variable. we can re-assign the value of the variable declared with let


  • Block Scope: Block scoped variable means it is accessible with the declared block area only. The block defines with the second bracket that we use for declaring any function or loop or any conditions.

  • Hoisting: accessing the let variable without declaring it will result ReferenceError.

const:The const keywords were added to JavaScript in 2015. we cannot re-assign the variable declared with const key word


  • Block Scope: it is also block scoped like let variable. it is not re-assignable variable but it doesn't mean that the value is immutable. value is mutable but we can to re-assign any value to the variable declared with const.

  • Hoisting: It is the same as let keyword.

Use case of Null and Undefined

There are six types of primitive Variables available in javascript. Such as Number, String, Boolean, NULL, Undefined and NaN.so NULL and Undefined also a type of primitive variable. let see some use case of null and undefined:

use case of null

  • Absence of a value: Null indicates the absence of value of an object or variable.
  • Clearing Values: NULL is also used to clear or reset the value of variables or properties. it helps to free up the memory by removing Reference of object.

use case of Undefined

  • Default Values: undefined is used as default value of function parameters
  • Properties and Variables Not Yet Defined: Accessing the properties or variables which is not declared yet will give undefined result.
  • No return statement: when a function does not have return statement. calling of that function will return undefined.
  • searching wrong array element: when accessing an element of an array that doesn't exist, it will give undefined result

Definition of REST API

REST APIs are widely used for building web services that enable different software systems to communicate over the internet. They are popular for their simplicity, scalability, and ease of integration, making them a fundamental part of modern web development and the foundation for many web-based applications and services. REST API stands for Representational State Transfer Application Programming Interface, is a type of web service that follows the principles and constraints of REST architectural style. It is a set of rules and conventions for designing and interacting with web-based services using HTTP (Hypertext Transfer Protocol).Here's a breakdown of what that means:

  • 1. Rules and Conventions: REST APIs have a specific way of doing things. They use a common language and a set of rules that both the program making requests (like a mobile app) and the program providing information (like a server) understand.

  • 2. Communication:Just like you communicate your food order to a waiter, software programs communicate with each other using HTTP, which is the same protocol your web browser uses to load web pages.

  • 3.Resources:REST APIs deal with resources, which can be thought of as pieces of data. These resources have unique addresses (like web page URLs) that programs can use to find and interact with them.

  • 4. Operations: You can think of REST API operations like actions you can perform on the resources. For example, you can ask for information about a resource (like asking for the menu in a restaurant), create new resources (like placing an order), update existing resources (like customizing your meal), or delete them (like canceling an order).

  • 5. Stateless: REST APIs don't remember anything about you once they've served your request. Each time you make a request, you need to include all the information the API needs to understand and fulfill that request.
